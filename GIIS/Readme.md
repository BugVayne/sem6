### Реализованные алгоритмы  

#### 1. **Bresenham (Брезенхем)**
- **Тип**: Целочисленный алгоритм  
- **Идея**: Использует только сложение и вычитание для минимизации вычислений.  
- **Особенности**:  
  - Работает только с целыми числами (эффективен для процессоров).  
  - Выбирает ближайшие пиксели к идеальной линии.  
  - Оптимизирован для осей X/Y (использует симметрию).  
- **Сложность**: O(max(Δx, Δy)).  

#### 2. **CDA (Digital Differential Analyzer)**
- **Тип**: Аппроксимационный алгоритм  
- **Идея**: Интерполирует координаты с фиксированным шагом.  
- **Особенности**:  
  - Использует вещественные числа для вычислений.  
  - Прост в реализации, но менее эффективен из-за операций с плавающей точкой.  
  - Рисует линию как последовательность точек с равномерным шагом.  
- **Сложность**: O(max(Δx, Δy)).  

#### 3. **Wu's Algorithm (Алгоритм Ву)**
- **Тип**: Алгоритм с антиалиасингом  
- **Идея**: Сглаживание линии за счет частичного закрашивания пикселей.  
- **Особенности**:  
  - Использует градации цвета для создания плавных линий.  
  - Работает с вещественными числами.  
  - Реализует два режима: "пологий" и "крутой" наклон.  
- **Сложность**: O(max(Δx, Δy)).  

### Интерактивность  
- Клик левой кнопкой - установка **начальной точки**  
- Второй клик - установка **конечной точки** → автоматическое построение линии  
- Привязка к сетке для точного позиционирования  

### Настройки визуализации  
- **Режим отладки**:  
  - Пошаговая отрисовка пикселей  
  - Вывод координат в консоль  
  - Замедленная визуализация (задержка 50 мс на шаг)  
- **Автоочистка холста** после построения (можно отключить)  
- Настраиваемый **размер пикселя** (по умолчанию 10px)

## 1. Реализованные алгоритмы

### Алгоритм Брезенхема для окружности
**Суть алгоритма:**
- Рисует окружность по заданному центру и радиусу
- Использует симметрию окружности для отрисовки только 1/8 части, остальные точки вычисляются зеркально
- Основная формула принятия решения: `d = 3 - 2*radius`
- На каждом шаге выбирает между восточным и юго-восточным пикселями
- Работает только с целочисленными операциями, что делает его эффективным

### Алгоритм Брезенхема для эллипса
**Суть алгоритма:**
- Разделяет эллипс на две области с разными условиями перехода
- В первой области (где наклон < 1) основной шаг по x
- Во второй области (где наклон ≥ 1) основной шаг по y
- Использует разные формулы для принятия решений в каждой области
- Также использует симметрию эллипса для отрисовки только 1/4 части

![image](https://github.com/user-attachments/assets/93880e13-4752-45fc-9bf6-a87f411e431b)


### Алгоритм для гиперболы
**Суть алгоритма:**
- Рисует ветви гиперболы относительно центра
- Разделяет отрисовку на две области аналогично эллипсу
- В первой области шагает по y, пока наклон касательной > 1
- Во второй области шагает по x
- Использует модифицированную формулу принятия решения

![image](https://github.com/user-attachments/assets/ecb9a709-62b0-45b7-ad74-b70503dd1ccf)


### Алгоритм для параболы
**Суть алгоритма:**
- Рисует параболу, направленную вверх или вниз
- Разделяет отрисовку на две области
- В первой области шагает по x, пока x² < 4py
- Во второй области шагает по y
- Использует симметрию относительно вертикальной оси

![image](https://github.com/user-attachments/assets/deeda571-0cdf-4a2a-9acf-517148158fea)


### Выбор кривой
- Пользователь может выбрать один из четырех типов кривых:
  - Окружность
  - Эллипс
  - Гипербола
  - Парабола
- Выбор осуществляется через кнопки в диалоговом окне

### Режимы работы
- **Debug Mode**: при включении выводит в консоль информацию о каждой отрисовываемой точке с небольшой задержкой
- **Clear Canvas After Drawing**: автоматически очищает холст перед отрисовкой новой фигуры

### Ввод данных
- Для каждой кривой требуется разное количество точек:
  - Окружность: 2 точки (центр и точка на окружности)
  - Эллипс: 3 точки (центр, конечная точка по x и конечная точка по y)
  - Гипербола: 2 точки (центр и точка для параметра a)
  - Парабола: 2 точки (вершина и точка для параметра p)
