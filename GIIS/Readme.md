### Реализованные алгоритмы  

#### 1. **Bresenham (Брезенхем)**
- **Тип**: Целочисленный алгоритм  
- **Идея**: Использует только сложение и вычитание для минимизации вычислений.  
- **Особенности**:  
  - Работает только с целыми числами (эффективен для процессоров).  
  - Выбирает ближайшие пиксели к идеальной линии.  
  - Оптимизирован для осей X/Y (использует симметрию).  
- **Сложность**: O(max(Δx, Δy)).  

#### 2. **CDA (Digital Differential Analyzer)**
- **Тип**: Аппроксимационный алгоритм  
- **Идея**: Интерполирует координаты с фиксированным шагом.  
- **Особенности**:  
  - Использует вещественные числа для вычислений.  
  - Прост в реализации, но менее эффективен из-за операций с плавающей точкой.  
  - Рисует линию как последовательность точек с равномерным шагом.  
- **Сложность**: O(max(Δx, Δy)).  

#### 3. **Wu's Algorithm (Алгоритм Ву)**
- **Тип**: Алгоритм с антиалиасингом  
- **Идея**: Сглаживание линии за счет частичного закрашивания пикселей.  
- **Особенности**:  
  - Использует градации цвета для создания плавных линий.  
  - Работает с вещественными числами.  
  - Реализует два режима: "пологий" и "крутой" наклон.  
- **Сложность**: O(max(Δx, Δy)).  

### Интерактивность  
- Клик левой кнопкой - установка **начальной точки**  
- Второй клик - установка **конечной точки** → автоматическое построение линии  
- Привязка к сетке для точного позиционирования  

### Настройки визуализации  
- **Режим отладки**:  
  - Пошаговая отрисовка пикселей  
  - Вывод координат в консоль  
  - Замедленная визуализация (задержка 50 мс на шаг)  
- **Автоочистка холста** после построения (можно отключить)  
- Настраиваемый **размер пикселя** (по умолчанию 10px)

## 1. Реализованные алгоритмы

### Алгоритм Брезенхема для окружности
**Суть алгоритма:**
- Рисует окружность по заданному центру и радиусу
- Использует симметрию окружности для отрисовки только 1/8 части, остальные точки вычисляются зеркально
- Основная формула принятия решения: `d = 3 - 2*radius`
- На каждом шаге выбирает между восточным и юго-восточным пикселями
- Работает только с целочисленными операциями, что делает его эффективным

### Алгоритм Брезенхема для эллипса
**Суть алгоритма:**
- Разделяет эллипс на две области с разными условиями перехода
- В первой области (где наклон < 1) основной шаг по x
- Во второй области (где наклон ≥ 1) основной шаг по y
- Использует разные формулы для принятия решений в каждой области
- Также использует симметрию эллипса для отрисовки только 1/4 части

![image](https://github.com/user-attachments/assets/93880e13-4752-45fc-9bf6-a87f411e431b)


### Алгоритм для гиперболы
**Суть алгоритма:**
- Рисует ветви гиперболы относительно центра
- Разделяет отрисовку на две области аналогично эллипсу
- В первой области шагает по y, пока наклон касательной > 1
- Во второй области шагает по x
- Использует модифицированную формулу принятия решения

![image](https://github.com/user-attachments/assets/ecb9a709-62b0-45b7-ad74-b70503dd1ccf)


### Алгоритм для параболы
**Суть алгоритма:**
- Рисует параболу, направленную вверх или вниз
- Разделяет отрисовку на две области
- В первой области шагает по x, пока x² < 4py
- Во второй области шагает по y
- Использует симметрию относительно вертикальной оси

![image](https://github.com/user-attachments/assets/deeda571-0cdf-4a2a-9acf-517148158fea)


### Выбор кривой
- Пользователь может выбрать один из четырех типов кривых:
  - Окружность
  - Эллипс
  - Гипербола
  - Парабола
- Выбор осуществляется через кнопки в диалоговом окне

### Режимы работы
- **Debug Mode**: при включении выводит в консоль информацию о каждой отрисовываемой точке с небольшой задержкой
- **Clear Canvas After Drawing**: автоматически очищает холст перед отрисовкой новой фигуры

### Ввод данных
- Для каждой кривой требуется разное количество точек:
  - Окружность: 2 точки (центр и точка на окружности)
  - Эллипс: 3 точки (центр, конечная точка по x и конечная точка по y)
  - Гипербола: 2 точки (центр и точка для параметра a)
  - Парабола: 2 точки (вершина и точка для параметра p)

## 1. Реализованные алгоритмы

### 1 Кривая Эрмита (Hermite Curve)
**Суть алгоритма:**
- Использует 4 контрольные точки: 
  - P1 - начальная точка кривой
  - P4 - конечная точка кривой
  - R1 - вектор касательной в начальной точке
  - R4 - вектор касательной в конечной точке
- Основан на матрице Эрмита, которая преобразует параметр t в координаты точки на кривой
- Формула: `P(t) = T * H * G`, где:
  - T = [t³, t², t, 1]
  - H - матрица Эрмита
  - G - матрица контрольных параметров
- Плавно интерполирует между точками, сохраняя заданные направления в начальной и конечной точках

![image](https://github.com/user-attachments/assets/b2d7f564-5ea2-4d89-99d4-a24a0fa1db55)


### 2 Кривая Безье (Bezier Curve)
**Суть алгоритма:**
- Может работать с любым количеством контрольных точек (≥4 в данной реализации)
- Использует полиномы Бернштейна для вычисления весов контрольных точек
- Формула: `P(t) = Σ (B(i,n,t) * P_i)`, где:
  - B(i,n,t) - полином Бернштейна
  - P_i - i-я контрольная точка
  - n - степень кривой (количество точек - 1)
- Кривая всегда проходит через первую и последнюю контрольные точки
- Обладает свойством выпуклой оболочки - кривая всегда лежит внутри фигуры, образованной контрольными точками

![image](https://github.com/user-attachments/assets/5b8d6440-407f-469e-9bb7-f6ff9777a4b3)


### 3 B-сплайн (B-Spline)
**Суть алгоритма:**
- Использует как минимум 4 контрольные точки (в реализации требуется ≥8)
- Основан на кубических базисных функциях
- Использует матрицу B для вычисления координат:
  `P(t) = T * B * G`, где:
  - T = [t³, t², t, 1]
  - B - матрица B-сплайна
  - G - матрица контрольных точек
- Кривая НЕ проходит через контрольные точки (кроме особых случаев)
- Обеспечивает более плавные переходы между сегментами кривой
- Локальное свойство - изменение одной контрольной точки влияет только на часть кривой

![image](https://github.com/user-attachments/assets/c6a6fbfd-eb8a-4326-abca-00e523be86d4)

## Алгоритмы заполнения (заливки)

### 1. Простой алгоритм заливки с затравкой
- **Тип**: Рекурсивный/стековый алгоритм
- **Принцип работы**:
  - Начинается с затравки (seed point)
  - Заливает все соседние пиксели того же цвета
  - Использует 4-связную область
- **Преимущества**:
  - Простота реализации
- **Недостатки**:
  - Может вызывать переполнение стека для больших областей
  - Неэффективен для сложных форм

### 2. Алгоритм заливки с затравкой по строкам
- **Тип**: Улучшенный алгоритм заливки
- **Принцип работы**:
  - Заливает целые строки пикселей
  - Запоминает верхние и нижние пиксели для дальнейшей обработки
  - Использует стек для хранения граничных пикселей
- **Преимущества**:
  - Более эффективен чем простой алгоритм
  - Меньше рекурсивных вызовов
- **Недостатки**:
  - Сложнее в реализации
![image](https://github.com/user-attachments/assets/adf3e148-20c7-4903-ba04-c6ba0761243f)
 

### 3. Алгоритм с упорядоченным списком ребер (Ordered Edge List)
- **Тип**: Алгоритм заполнения многоугольников
- **Принцип работы**:
  - Создает список всех ребер многоугольника
  - Сортирует ребра по минимальной y-координате
  - Использует активный список ребер для заполнения
- **Преимущества**:
  - Эффективен для сложных многоугольников
  - Корректно обрабатывает самопересечения
- **Недостатки**:
  - Требует предварительной обработки ребер

### 4. Алгоритм с активным списком ребер (Active Edge List)
- **Тип**: Алгоритм заполнения многоугольников
- **Принцип работы**:
  - Поддерживает список "активных" ребер для текущей строки сканирования
  - Обновляет список при переходе к следующей строке
  - Заполняет пиксели между парами ребер
- **Преимущества**:
  - Эффективен для больших многоугольников
  - Минимизирует вычисления
- **Недостатки**:
  - Требует правильной сортировки ребер

## Алгоритмы построения выпуклой оболочки

### 1. Алгоритм Грэхема (Graham's Scan)
- **Принцип работы**:
  - Находит точку с минимальной y-координатой
  - Сортирует все точки по полярному углу относительно начальной
  - Строит оболочку, отбрасывая точки, создающие невыпуклость
- **Сложность**: O(n log n)
- **Преимущества**:
  - Эффективен для больших наборов точек
- **Недостатки**:
  - Требует сортировки

### 2. Алгоритм Джарвиса (Jarvis March или Gift Wrapping)
- **Принцип работы**:
  - Находит самую левую точку
  - На каждой итерации находит точку с минимальным полярным углом
  - Продолжает, пока не вернется в начальную точку
- **Сложность**: O(nh), где h - число точек в оболочке
- **Преимущества**:
  - Прост в реализации
  - Эффективен для малого числа точек в оболочке
- **Недостатки**:
  - Медленный для больших наборов точек

## Реализация

### Триангуляция Делоне
- **Назначение**: Разбивает множество точек на треугольники, оптимизируя форму
- **Ключевое свойство**: Ни одна точка не попадает внутрь описанных окружностей треугольников
- **Алгоритм**:
  1. Создание супертреугольника, охватывающего все точки
  2. Поочередное добавление точек с перестроением триангуляции
  3. Удаление треугольников, связанных с супертреугольником
- **Сложность**: O(n²) в данной реализации
![image](https://github.com/user-attachments/assets/b046de86-6309-4db6-9466-6d94d10953e0)

### Диаграмма Вороного
- **Назначение**: Разбивает плоскость на ячейки ближайших точек
- **Алгоритм**:
  1. Построение триангуляции Делоне
  2. Соединение центров описанных окружностей соседних треугольников
- **Применение**: Анализ ближайшего соседства, зоны влияния
